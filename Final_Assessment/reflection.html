WEB222 Final Assessment Winter 2022

• Explain the concept of Responsive Design. What does the term mean?

Responsive web design is that the design and development of pages should respond to user behaviour and equipment environment (system platform, screen size, screen orientation, etc.). Whether the user is using a laptop or iPad, our page should be able to switch the resolution automatically, image size and related script functions to adapt to different devices.

• How does responsive design relate to web technologies?

  1. Media Queries
    Media Queries allow us to run a series of tests (e.g. whether the user's screen is greater than a certain width or a certain resolution) and apply CSS selectively to style the page appropriately for the user's needs.

  2. Flexible grids
    A flexible grid means that you don't need to target every possible device size and build a pixel perfect layout for it. Using a flexible grid, you only need to add in a breakpoint and change the design when the content starts to look bad.

  3. Modern layout technologies
    Modern layout methods such as Multiple-column layout, Flexbox, and Grid are responsive by default. They all assume that you are trying to create a flexible grid and give you more accessible ways to do so. For example:
    a) Multicol   b) Flexbox   c) CSS grid   d) Responsive images   e) Responsive typography   …

• What did you learn about building web apps for different devices through your research?

I learned to use Flexbox, Responsive images, and CSS Grid Layout to build web apps for different devices.

  1. In Flexbox, flex items will shrink and distribute space between the items according to the space in their container. By changing the values for flex-grow and flex-shrink, we can indicate how we want the items to behave.

  2. Responsive Images allow designers to create responsive layouts that serve different image sizes at different resolutions.

  3. In CSS Grid Layout, the one unit allows the distribution of available space across grid tracks.

• Describe your learning experience in the course

In web222, I have learned the basic knowledge of JavaScript, HTML and CSS. It's not difficult to get in touch initially, but after in-depth study, I find that they are so sophisticated.

In week 1 and 2, I learned JavaScript basic syntax, built‐in functions, and user-defined functions. I think it’s simple for me.

In week 3 and 4, I learned JavaScript String, Array and customized objects. It’s a little complicated. I tried my best but couldn’t say I knew it very well.

In week 5 and 6, I learned how to write HTML Containers / Media Elements and update the DOM with data. I could understand this entry part.

In week 7 and 8, I learned DOM/JavaScript coding, dynamically adding text heading, populating HTML tables using DOM and generating images using DOM (rather than innerHTML property). It’s more complicated. I could cope with the assignment.

In week 9 and 10, I learned about styling elements, page layout and image gallery with CSS. With studying assignment 5, I read some other people’s HTML layout code and CSS code. I found I only knew the surface of web technology. People could present glorious web pages by JS, HTML and CSS. I'm still far from them. Especial for responsive web design, I knew I should “make content like water”. But it’s really really hard for me.

From week 11 to the end, I learned the introduction to Client-side validation by HTML5 or JS. I’m okay with this part, but not sure I could deal with more requirements and a more complex environment.
